## [2026-05-28] - Phase 1H Completed: 0.1.0 Scope Realignment

### Context

Phase 1G completed a technical MVP validated on emulator API 36 according to the limited-access policy: home, gallery, import, preview cache, filters, compare, JPEG/PNG export, gallery saving, metadata scrub re-encode, no `INTERNET`, Windows preview, and circumscribed limited access.

The user clarified that this is not enough for a **0.1.0 stable** release. The 0.1.0 milestone is therefore redefined to have a more complete product scope.

### Mandatory 0.1.0 Requirements Recorded

- Real `Take photo`.
- `Open photo`.
- Real `More photos`.
- Editor with filter stack.
- `Clear all filters`.
- More readable and calibrated filters.
- Clear filter miniature previews.
- Non-invasive compare via toggle or swipe.
- Removal of the drag-to-compare text.
- JPEG/PNG export.
- Metadata toggle in the export bottom sheet.
- No backend, login, tracking, ads, Firebase, machine learning, or `INTERNET`.

### Technical Decisions Recorded

- Camera: Prefer official Flutter `camera` package.
- `android.permission.CAMERA` will become an expected permission when the camera is implemented.
- `android.permission.RECORD_AUDIO` remains forbidden.
- `image_picker` is not the main shortcut unless a new explicit decision is made.
- `More photos`: 0.1.0 direction with multi-select, shared stack, and multiple export.
- Filter stack: Ordered list, export of the full stack, mandatory `Clear all`.
- Compare: Non-invasive toggle/swipe; export independent of compare mode.
- Metadata toggle: Default ON; OFF unavailable until real metadata preservation exists.
